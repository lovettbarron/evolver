## Cross-Phase Integration

### Wiring Status: 27/29 requirement integration paths confirmed

All major cross-phase data flows are properly wired:
- Phase 7 → 8: InstrumentConfigSchema/loadInstrumentConfig consumed by module routes
- Phase 7 → 9: PatchSchema stubs (cable_routing, knob_settings) filled by Phase 9 schemas
- Phase 9 → 11: Curriculum patches follow Phase 9 schema conventions
- Phase 10+11 → content pipeline: 25 sessions + 23 patches bundled and served
- Phase 12 → 13: Panel architecture pattern (metadata, SVG, tooltips, integrations) replicated for Cascadia

### Integration Issues

**1. Cascadia Patch Panel View — Cable ID Mismatch (MULTI-04)**
- `patch-detail.tsx` transforms cable routing labels (e.g., "LFO X OUT") to jack IDs via kebab-case (`jack-lfo-x-out`)
- Actual `CONTROL_METADATA` keys use full module prefixes (`jack-lfo-xyz-x-out`)
- Result: CascadiaPanel mounts but cable bezier paths don't render
- Fix: Add reverse lookup from display names to CONTROL_METADATA keys

**2. Standalone Panel Route — Missing instrumentSlug (Minor)**
- `/instruments/[slug]/panel/page.tsx` doesn't pass `instrumentSlug` to `StickyHeader`
- Quick Ref drawer on standalone panel page won't show the Panel tab
- Non-blocking: the page IS the panel
