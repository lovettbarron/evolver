## Spacing Scale

| Token | Value | Usage |

## Layout Contract

### Landing Page (Instrument Selector)

Replaces the current single-instrument hero card.

- Centered layout, max-width 720px
- Page title: "Choose Your Instrument" -- Display size (36px), bold, centered
- Subtitle: "Pick up where you left off" -- Body size (16px), `--muted`, centered, below title with `md` spacing
- Instrument cards: horizontal row on desktop (gap `lg`), vertical stack on mobile (gap `lg`)
- Cards equally sized: `flex-1` within row, `min-width: 280px`, `max-width: 360px`
- Vertical spacing: `3xl` above title, `2xl` between subtitle and cards, `2xl` below cards

### Instrument Card

- Background: `--surface`
- Padding: `2xl` (48px)
- Border radius: 6px (matches existing card pattern from HeroCard)
- Content stack (vertical, gap `sm`):
  1. Display name: 24px bold `--text`
  2. Tagline: 14px regular `--muted`
  3. Stats row: "{N} sessions / {N} patches" -- 14px JetBrains Mono `--param`
  4. CTA link: "Explore {instrument}" -- styled as accent text link with underline, not a button
- Hover state: border 1px `--accent` (transition 150ms)
- Entire card is clickable (wraps in Link to `/instruments/{slug}`)
- Cards should feel like "mission briefings" per CONTEXT.md specifics

### Nav (Instrument Switcher)

The hardcoded nav links are replaced with a dynamic instrument switcher.

- Brand text "evolver" stays left-aligned (project name, not instrument name) with demo badge
- Instrument switcher positioned after brand text: shows current instrument display_name
- Switcher trigger: current instrument name in `--muted` text, 14px, with chevron-down icon (lucide `ChevronDown`, 14px)
- Dropdown: `--surface` background, 1px border `color-mix(in srgb, var(--color-surface) 50%, var(--color-muted))`, border-radius 6px, shadow none (editorial aesthetic -- no elevation)
- Dropdown items: 14px `--text`, padding `sm` vertical / `md` horizontal, min-height 44px
- Active instrument: `--accent` left border 2px on the item
- Clicking an instrument navigates to `/instruments/{slug}` and closes dropdown
- Sub-links (Sessions, Patches, MIDI, Progress) remain right-aligned, scoped to selected instrument
- Sub-links conditionally rendered: hide "MIDI" when instrument has `sysex: false`
- Sub-links hide "Progress" when no sessions exist (future-proofing for new instruments with no curriculum yet)
- Home link always present

### Nav Dropdown Behavior

- Trigger: click (not hover) on instrument name
- Close: click outside, click on item, Escape key
- Position: below trigger, left-aligned with trigger text
- Width: auto, min-width 200px
- z-index: 40 (below modals at 50, per confirm-dialog pattern)

### No-SysEx Capability Page (`/instruments/{slug}/midi`)

Generic informational page shown when `sysex: false` in instrument config.

- Centered layout, max-width 480px
- Icon: `AlertCircle` from lucide-react, 48px, `--muted` color
- Heading: "{instrument} doesn't support MIDI SysEx" -- 24px bold
- Body: "This instrument uses CV/gate patching instead of MIDI program memory. Patches are documented with cable routing and knob positions." -- 16px `--muted`
- CTA: "Browse {instrument} Patches" -- accent text link to `/instruments/{slug}/patches`
- Vertical spacing: `3xl` top padding, `lg` between elements

### Instrument Overview Page Changes

- Receives the "next session" hero card from the landing page
- Hero card appears at top of instrument overview when sessions exist
- Below hero card: existing instrument overview content (architecture, signal flow, links)
- Reference PDFs read from `instrument.json` `reference_pdfs` array instead of hardcoded

## Accessibility Contract

- Instrument switcher dropdown: `role="listbox"`, items have `role="option"`, `aria-selected` for active instrument
- Switcher trigger: `aria-haspopup="listbox"`, `aria-expanded` state
- Keyboard navigation: Arrow keys move between dropdown items, Enter selects, Escape closes
- Focus trap in dropdown when open; focus returns to trigger on close
- Instrument cards: entire card clickable via wrapping `<a>` (not onClick on div)
- No-SysEx page: uses semantic `<main>` with descriptive heading (no modal, no trap)
- Conditional nav links: hidden via `display: none` (not `visibility: hidden`) so screen readers skip them
- All existing accessibility contracts from Phase 2 remain in effect
