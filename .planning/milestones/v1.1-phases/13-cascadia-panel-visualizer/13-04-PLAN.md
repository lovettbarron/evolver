Add `data-cascadia-panel` marker divs to each session at points where the learner interacts with specific controls. Follow the same placement pattern as Evolver sessions (read sessions/evolver/03-analog-osc-waveshapes.md for the pattern): place markers AFTER the text describes what to do, so the panel appears as a visual reference for the instructions above it.

**Marker placement guidelines:**
- Place 1-3 markers per session (same density as Evolver sessions)
- First marker: after the initial setup/orientation instructions, showing the starting module(s)
- Additional markers: at key exercise steps where the learner adjusts different controls or patches cables
- Use data-sections to zoom to the relevant module(s)
- Use data-knobs for any specific knob/slider values mentioned in the session text
- Use data-highlights to call attention to controls being discussed (blue for "adjust this", amber for "observe this")
- Use data-cables for sessions that involve patching (sessions 6+ typically have cable exercises)

**Per-session marker content:**

The control IDs in markers MUST match the IDs in cascadia-panel-data.ts CONTROL_METADATA. Read the file to get exact IDs.

Session 1 (First Sound): data-sections="vco-a,mixer,vcf,output-control" — overview of the basic signal path. Highlight the PITCH knob and VOLUME knob. No cables (normalled path).

Session 2 (PWM/Sub): data-sections="vco-a" — highlight PW slider and PW MOD slider. Show PULSE POSITION switch.

Session 3 (Filter/Wavefold/FM/FX): Multiple markers — one for VCF section, one for wave-folder section, one for fx-send-return. Highlight CUTOFF, RESONANCE, FOLD controls.

Sessions 4-6 (Oscillators): Focus on vco-a, vco-b, wave-folder sections. Session 6 (Wave Folder) is first session with cables — add data-cables for the patching exercise (e.g., VCO A SAW to FOLD IN).

Sessions 7-9 (Envelopes): Focus on envelope-a, envelope-b, vca-a, vca-b-lpf sections. Include ADSR knob values from the session text.

Sessions 10-12 (Filters): Focus on vcf, wave-folder, vca-b-lpf. Session 12 (LPG) involves cables — add data-cables.

Session 13 (LFO Deep Dive): Focus on lfo-xyz with data-cables showing LFO routing to various destinations.

```markdown
### Cascadia Panel Markers

Cascadia sessions use `data-cascadia-panel` markers with the same attribute format as Evolver, plus `data-cables` for visualizing patch cable connections.

#### Basic Format
\`\`\`html
<div data-cascadia-panel
  data-sections="vco-a,mixer"
  data-knobs="knob-vco-a-pitch:64,slider-mixer-vco-a:100"
  data-highlights="knob-vco-a-pitch:blue,jack-vco-a-saw:amber"
></div>
\`\`\`

#### With Cable Connections
\`\`\`html
<div data-cascadia-panel
  data-sections="vco-a,mixer,wave-folder"
  data-highlights="jack-vco-a-saw:amber,jack-wave-folder-in:blue"
  data-cables="jack-vco-a-saw>jack-wave-folder-in:audio"
></div>
\`\`\`

#### data-cables Format
Comma-separated entries: `sourceJackId>destJackId:signalType`
- Signal types: `audio`, `cv`, `mod`, `default`
- Jack IDs match CONTROL_METADATA keys in cascadia-panel-data.ts
- Connected jacks automatically receive highlight glow
- Cable paths render as colored bezier curves on the panel

#### Section Names
midi-cv, vco-a, vco-b, envelope-a, envelope-b, line-in, mixer, vcf, wave-folder, vca-a, push-gate, utilities, lfo-xyz, patchbay, vca-b-lpf, fx-send-return, output-control
```
