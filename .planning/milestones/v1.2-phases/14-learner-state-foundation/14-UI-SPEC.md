## Component Inventory

### 1. Completion Toggle — Sticky Bottom Bar

**Location:** Session detail page (`/instruments/[slug]/sessions/[session]`)
**Type:** Client component (`'use client'`)
**Behavior:** Sticky to viewport bottom. Always visible without scrolling. Single tap toggles completion state. No confirmation dialog (D-02).

| Property | Specification |
|----------|--------------|
| Position | `fixed` bottom-0, full width, z-10 |
| Height | 56px (16px top/bottom padding + 24px content) |
| Background | `--color-bg` with top border `1px solid` at `--color-surface` |
| Layout | Flex row, items-center, justify-between, max-w matching page content, centered |
| Left side | Checkbox + session title |
| Right side | None (keep it minimal) |

**Checkbox spec:**

| State | Visual |
|-------|--------|
| Unchecked | 20x20px rounded-[4px] box, 2px border `--color-muted`, empty interior |
| Checked | 20x20px rounded-[4px] box, filled `--color-accent`, Lucide `Check` icon 14px in `--color-bg` |
| Hover (unchecked) | Border brightens to `--color-text` |
| Transition | `transition-colors duration-150` on background and border |

**Label spec:**

| State | Visual |
|-------|--------|
| Unchecked | "Mark complete" in 14px `--color-muted` weight 400 |
| Checked | "Completed" in 14px `--color-accent` weight 400 |

**Accessibility:**
- `role="checkbox"` with `aria-checked` reflecting state
- `aria-label="Mark session {title} as complete"` / `"Session {title} completed"`
- Keyboard: Space/Enter toggles

**Demo mode (D-07):** Entire sticky bar is hidden (`display: none`). Do not render the component at all in demo mode.

### 2. Resume Bar

**Location:** Instrument home page (`/instruments/[slug]`), positioned as the first content element below the instrument header.
**Type:** Client component consuming Zustand store + server-passed props.

| Property | Specification |
|----------|--------------|
| Background | `--color-surface` |
| Border radius | 8px |
| Padding | 24px (--spacing-lg) |
| Layout | Flex row, items-center, gap-md, justify-between |
| Max width | Match parent content width |

**Content — has next session (D-05):**

| Element | Specification |
|---------|--------------|
| Left icon | Lucide `BookOpen` at 20px in `--color-muted` |
| Label | "Continue where you left off" in 14px `--color-muted` weight 400 |
| Session title | Session number + title in 16px `--color-text` weight 400 |
| Right arrow | Lucide `ArrowRight` at 20px in `--color-accent` |

Label and session title stack vertically (flex-col), arrow sits on the right.

**Content — all complete (D-06):**

| Element | Specification |
|---------|--------------|
| Left icon | Lucide `CheckCircle` at 20px in `--color-accent` |
| Label | "All sessions complete" in 14px `--color-accent` weight 400 |
| Link | "Explore patches and challenges" in 16px `--color-text` weight 400, underline on hover |
| Right arrow | Lucide `ArrowRight` at 20px in `--color-muted` |

**Content — no sessions visited yet (empty state):**

| Element | Specification |
|---------|--------------|
| Left icon | Lucide `Play` at 20px in `--color-accent` |
| Label | "Ready to start?" in 14px `--color-muted` weight 400 |
| Session title | First session number + title in 16px `--color-text` weight 400 |
| Right arrow | Lucide `ArrowRight` at 20px in `--color-accent` |

**Hover state:** Entire bar gets `bg-[#1e1e1e]` (slightly lighter than surface). Cursor pointer. Transition 150ms.

**Click behavior:** Entire bar is a link wrapping to the target session URL.

**Demo mode (D-07):** Resume bar renders in read-only mode showing the next session in the synthetic journey. Visually identical, but state comes from synthetic data.

**Accessibility:**
- Rendered as `<a>` or `<Link>` wrapping the full bar
- `aria-label="Continue to session {number}: {title}"` or `"All sessions complete — explore patches and challenges"`

### 3. Scroll Compensation

Session detail pages must add `padding-bottom: 72px` (56px bar + 16px breathing room) to prevent the sticky bar from covering bottom content.

## Interaction States

### Completion Toggle State Machine

```
UNCHECKED ──(tap)──> CHECKED
CHECKED   ──(tap)──> UNCHECKED
```

- Toggle is synchronous to Zustand store (instant visual feedback)
- Persist middleware writes to localStorage asynchronously
- No loading spinner, no optimistic UI needed (localStorage is synchronous in practice)

### Resume Bar State Machine

```
LOADING ──(hydration complete)──> HAS_NEXT | ALL_COMPLETE | FIRST_SESSION
```

- During SSR/hydration: render nothing (avoid flash of wrong state)
- After hydration: show the correct variant based on merged completion data
- Hydration strategy: mount with `display: none`, reveal after Zustand hydration completes (use `onRehydrateStorage` callback)

### Demo Mode

- Completion toggle: not rendered
- Resume bar: rendered with synthetic data, not interactive beyond navigation
