Declared values (multiples of 4, inherited from project globals.css):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between dot indicators |
| md | 16px | Default element spacing, card internal padding |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.6 |
| Label | 14px | 400 | 1.4 |
| Heading | 24px | 700 | 1.2 |
| Display | 48px | 700 | 1.1 |

Source: Body line-height 1.6 from globals.css `body`. Heading sizes from `.prose h2` (24px) and `.prose h3` (20px). Display size (48px) used for large additive count numbers on the progress dashboard. Label (14px) used for count labels, module names, and muted metadata.

Phase-specific usage:
- **Display (48px/700)**: The four additive count numbers (sessions, patches, modules, challenges)
- **Label (14px/400)**: Count labels below numbers ("Sessions Completed", "Patches Created", "Modules Done", "Challenges Completed")
- **Heading (24px/700)**: Section headings ("Your Progress", "Module Journey")
- **Body (16px/400)**: Empty state copy, challenge callout text

## Component Inventory

### New Components

#### ProgressDashboard
- **Route**: `/instruments/[slug]/progress`
- **Layout**: Single-column, max-width 640px, centered. Consistent with session-detail content width
- **Server component** for data loading, no client interactivity needed

#### CountCard
- **Structure**: Vertical stack — large number (Display/48px/700, accent color) above label (Label/14px/400, muted color)
- **Layout**: 4 cards in a horizontal row on desktop (gap-xl), 2x2 grid on mobile (gap-md)
- **Background**: `bg-surface`, padding `p-lg`, rounded `rounded-lg`
- **States**: Zero shows "0" (never hide cards), number always visible
- **Accessible**: `aria-label="{count} {label}"` on each card for screen readers

#### ModuleJourney
- **Structure**: Section heading "Module Journey" (Heading/24px/700) + row of 10 dots + module names below
- **Dots**: 12px diameter circles, `bg-accent` when complete, `bg-surface border border-muted` when incomplete
- **Module labels**: Below each dot, Label/14px/400, muted color. Show module number only ("1" through "10") to keep compact
- **Tooltip on hover**: Full module name appears on hover/focus (CSS title attribute, not a custom tooltip component)
- **Container**: `bg-surface` panel, padding `p-lg`, rounded `rounded-lg`
- **Responsive**: Dots and labels stay in a single horizontal row; 8px gap between dots. On narrow screens (< 480px) the row scrolls horizontally inside the panel

#### EmptyProgressState
- **Displayed when**: Zero sessions completed AND zero patches created
- **Structure**: Centered content block inside the progress page area
- **Heading**: "Start your first session" (Heading/24px/700, text color)
- **Body**: "Your progress appears here as you work through the curriculum. Every session counts." (Body/16px/400, muted color)
- **CTA**: "Start Session 1" button — same style as hero-card CTA (`bg-accent text-bg font-bold py-md px-xl rounded min-h-[48px]`), links to `/instruments/evolver/sessions/01-oscillators-intro`
- **No illustration**: Text-only, consistent with the editorial minimal aesthetic

#### ChallengeCallout (markdown rendering extension)
- **Trigger**: `> [!challenge]` callout blocks in session markdown
- **Structure**: Left-bordered box (matching existing `.callout` pattern)
- **Border**: 3px left border, `accent` color (same as existing callouts but distinguishable via icon/header)
- **Background**: `bg-surface`
- **Header**: Bold "Challenge" label with a target icon (lucide `Target` icon, 16px, accent color) inline before the label text
- **Body**: Challenge description text (Body/16px/400) including wikilink to reference patch
- **Padding**: `p-md` (matching existing callout padding)
- **Border radius**: `0 6px 6px 0` (matching existing callout pattern)

## Interaction States

### Progress Page Load
1. Server component fetches sessions, patches, daily note scan results
2. Counts render immediately (no loading skeleton needed — server-rendered)
3. If vault path configured: scan daily notes for `#session-XX` tags
4. If no vault path (demo mode): use synthetic completion data

### Module Dot Interaction
- **Hover/Focus**: Title tooltip shows full module name (e.g., "Module 3: Envelopes + LFOs")
- **No click action**: Dots are display-only, not navigable (the session browser handles navigation)
- **Keyboard**: Dots are not focusable (decorative, not interactive). Module names provide sufficient context

### Challenge Callout in Sessions
- **Static display**: Callout renders inline in session markdown flow. No interactive expand/collapse
- **Wikilink navigation**: `[[patch-name]]` renders as a link to the patch detail page (already supported by rendering pipeline)
- **No inline completion indicator**: Challenge completion is reflected on the progress page, not inline in the session

### Empty State to First Action
- Single CTA "Start Session 1" links directly to the first session
- After completing even one session, the empty state is replaced by the full dashboard with counts
