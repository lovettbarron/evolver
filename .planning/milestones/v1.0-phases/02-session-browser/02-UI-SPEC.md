## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, tag padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, content margins |
| xl | 32px | Layout gaps between major elements |
| 2xl | 48px | Major section breaks, hero spacing |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions: Touch targets for mobile navigation and CTA buttons use minimum 48px height (accessibility — uses 2xl token, exceeds WCAG 2.5.5 minimum).

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 (regular) | 1.6 | Inter |
| Label / Param | 14px | 400 (regular) | 1.5 | Inter (labels), JetBrains Mono (param values) |
| Heading | 24px | 700 (bold) | 1.2 | Inter |
| Display | 36px | 700 (bold) | 1.1 | Inter |

**Notes:**
- 4 sizes, 2 weights. Label and Param share the 14px size; font family (Inter vs JetBrains Mono) is the differentiator.
- Sans-serif headers (bold) and body per CONTEXT.md decision.
- Monospace parameter values get the `--param` color token to further distinguish them from labels.
- Display size used for the next-session hero card title and module headers.
- Body line-height at 1.6 for long-form session reading (Domus editorial feel).

## Color

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg` | `#0a0a0a` | Page background, app shell |
| Secondary (30%) | `--surface` | `#161616` | Cards, hero card, slide-out panel, sticky header |
| Text | `--text` | `#e8e8e8` | Body text, headings |
| Muted | `--muted` | `#737373` | Secondary text, module labels, source references, durations |
| Accent (10%) | `--accent` | `#c8ff00` | See reserved-for list below |
| Param | `--param` | `#a3e635` | Monospace parameter values (distinct from accent, softer) |
| Destructive | n/a | n/a | No destructive actions in this phase |

**Accent reserved for:**
- "Start" CTA button on the next-session hero card
- Active/current session indicator in the browse list
- Prev/next navigation arrows on hover
- Quick-reference card trigger button in sticky header

**Accent is NOT used for:** body links, module headers, session titles, or general interactive elements. Links use `--text` with underline on hover.

**Dark editorial palette rationale:** Inspired by Ghostly International (near-black, restrained, music-industry) and Domus magazine (editorial whitespace). The yellow-green accent (`#c8ff00`) is a deliberate high-contrast choice against near-black -- it reads as "synthesizer" and creates immediate visual hierarchy for the single most important action on any screen.

## Layout Contract

### Content Column
- Max width: 720px for session reading (CONTEXT.md decision)
- Centered with auto margins
- Horizontal padding: 24px (lg) on mobile, 32px (xl) on desktop

### Breakpoints
- Mobile: < 640px (single column, hamburger nav if needed)
- Tablet: 640px-1024px (content column, side padding increases)
- Desktop: > 1024px (full layout, slide-out panel has room)

### Page Templates

**Landing (Home)**
- Next-session hero card: centered, max-width 480px, prominent "Start" button
- Below hero: "Browse All Sessions" text link, instrument name link
- Generous vertical spacing (3xl above and below hero)

**Session List (`/instruments/[slug]/sessions`)**
- Grouped list under module headers (no accordion, all visible)
- Module header: Display size, bold, full-width divider line below
- Session row: number + title left-aligned, duration right-aligned, full-width clickable area
- No status indicators (deferred to Phase 5)

**Session Detail (`/instruments/[slug]/sessions/[id]`)**
- Sticky header: 48px height, `--surface` background, contains back arrow + session identifier + quick-ref button
- Single column, 720px max, long-scroll
- Content sections: warm-up, exercises (with parameter tables), output checklist
- Parameter tables: monospace values, `--param` color, `--surface` background for table rows
- Source references: inline markers styled as superscript in `--muted`, click reveals tooltip
- Bottom: prev/next session navigation (crosses module boundaries)

**Instrument Overview (`/instruments/[slug]`)**
- Instrument name as Display heading
- Architecture summary as body text
- Signal flow diagram (Mermaid, rendered inline)
- Links to: basic patch card, signal flow detail, "Start Curriculum" CTA

**Quick-Reference Slide-Out**
- Panel slides from right edge, 380px wide on desktop, full-width on mobile
- `--surface` background, 24px padding
- Contains: basic patch parameter table or signal flow content
- Dismiss via close button or click outside
- Content stays loaded (no re-fetch on toggle, per CONTEXT.md)

**Framework / About Method**
- Accessible from footer link
- Simple long-form page with `framework/` content rendered
- Not in main navigation

## Accessibility Contract

- All interactive elements reachable via keyboard (Tab order follows visual order)
- Focus indicators: 2px ring in `--accent` with 2px offset (visible against `--bg`)
- Slide-out panel: focus trap when open, Escape key dismisses
- Icon-only buttons (back arrow, close panel) have explicit `aria-label` attributes (see Copywriting Contract)
- Color contrast: `--text` (#e8e8e8) on `--bg` (#0a0a0a) = ratio 17.4:1 (exceeds WCAG AAA)
- Color contrast: `--muted` (#737373) on `--bg` (#0a0a0a) = ratio 5.2:1 (exceeds WCAG AA)
- Color contrast: `--accent` (#c8ff00) on `--bg` (#0a0a0a) = ratio 14.8:1 (exceeds WCAG AAA)
- Parameter tables use `<th>` elements with proper scope attributes
- Sticky header does not obscure focused content (scroll-padding-top matches header height)
