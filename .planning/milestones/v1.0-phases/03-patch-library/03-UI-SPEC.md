## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, card internal gaps |
| md | 16px | Default element spacing, card body padding |
| lg | 24px | Section padding, card outer padding |
| xl | 32px | Layout gaps between card grid rows |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets use min-h-[48px] (established pattern from hero-card.tsx and nav.tsx).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

| Label / Badge / Meta | 14px | 400 | 1.4 |
| Card Title | 20px | 700 | 1.2 |

## Color

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | --color-bg | #0a0a0a | Page background |
| Secondary (30%) | --color-surface | #161616 | Patch cards, sticky header, callout boxes, parameter table even rows |
| Text | --color-text | #e8e8e8 | Card titles, body text, active nav links |
| Muted | --color-muted | #737373 | Metadata text, inactive pills, type group headers, table headers |
| Accent (10%) | --color-accent | #c8ff00 | Active filter pill background, type badge text, callout left border, hover states |
| Param | --color-param | #a3e635 | Parameter values in tables (mono font) |

Accent reserved for:
- Active filter pill background (text becomes --color-bg on accent background)
- Type badge text color on cards
- Callout box left border (3px)
- Link hover state
- Focus ring color
- NOT per-type colors (all types use the same accent -- CONTEXT.md locked decision)

Destructive: Not applicable for this phase (no destructive actions).

## Component Inventory

### Patch Card

```
+----------------------------------+
| TYPE BADGE          2026-01-15   |  <- 14px uppercase, accent text / muted date
| Patch Name                       |  <- 20px bold
| Description snippet truncated    |  <- 16px text, 2 lines max
| to two lines maximum...          |
| #tag1  #tag2  #tag3              |  <- 14px muted
+----------------------------------+
```

- Background: surface
- Border-radius: 6px (matches existing code block and callout radius)
- Padding: lg (24px)
- Hover: border transitions to accent (same pattern as quick-ref button)
- Border default: 1px solid transparent
- Entire card is a clickable link (wraps in Next.js Link)
- Description clamped to 2 lines via line-clamp-2
- Tags displayed inline, separated by 8px gap, text color muted

### Filter Pills

```
[ All ] [ Bass ] [ Lead ] [ Pad ] [ Texture ]
```

- Container: horizontal flex, gap-sm (8px), overflow-x-auto on mobile
- Active pill: bg-accent, text-bg, font-weight 600, rounded-full, px-md py-xs
- Inactive pill: bg-transparent, text-muted, border 1px solid muted/30, rounded-full, px-md py-xs
- Hover (inactive): text-text, border-text
- Min touch target: min-h-[36px] (compact row, not primary CTA)
- Empty types hidden from pill list (CONTEXT.md locked decision)

### Type Group Header (All view only)

```
BASS ────────────────────────────
```

- Text: 14px uppercase tracking-wider, color muted
- Divider: 1px solid muted/20 extending to full width
- Margin-top: 2xl (48px) for first group, xl (32px) between groups
- Hidden when a specific type filter is active (CONTEXT.md locked decision)

### Card Grid Layout

- CSS Grid: grid-cols-1 on mobile, grid-cols-2 at min-width 640px
- Gap: lg (24px)
- Max-width: 960px (same content width as session detail prose)
- Centered with mx-auto

### Patch Detail - Sticky Header

Reuses existing `sticky-header.tsx` pattern with modifications:

- Back link text: "Patches" (href to `/instruments/[slug]/patches`)
- Center: patch name (truncated)
- Right: no Quick Ref button (not applicable for patches)

### Patch Detail - Metadata Block

```
BASS  -  Created in Session 06: Oscillator Mixing
```

- Type in uppercase, 14px, accent color
- Separator: " - " in muted
- Session link: 14px, text color, underline, hover accent
- Margin-bottom: lg (24px)

### Patch Detail - Parameter Sections

Parameter tables grouped by synth section with section headers:

```
Oscillators
| Parameter    | Value |
|-------------|-------|
| Osc 1 Freq  | 32    |
| Osc 1 Wave  | Saw   |

Filter
| Parameter    | Value |
|-------------|-------|
| Cutoff       | 80    |
```

- Section headers: 20px bold (h3 level), margin-top 1.5em
- Tables reuse existing .param-table CSS (already styled in globals.css)
- Param names in sans, param values in mono with --color-param

### Patch Detail - Callout Boxes (Playing Tips / Technique Notes)

Reuses existing .callout CSS:
- Background: surface
- Left border: 3px solid accent
- Padding: md (16px)
- Border-radius: 0 6px 6px 0
- Callout title (e.g. "Playing Tips"): 14px uppercase tracking-wide, muted, margin-bottom sm

### Patch Detail - Body Content

Rendered via existing `renderMarkdown()` pipeline inside `.prose` container. No new markdown rendering features needed.

## Interaction States

### Filter Pill Selection
- Click pill: updates URL search param `?type=bass` (enables shareable filtered URLs)
- "All" resets to no search param
- Transition: none (instant re-render, data is already loaded server-side)

### Card Hover
- Border color transitions from transparent to accent over 150ms
- Cursor: pointer (entire card is link)

### Empty Filter State
- When a type filter returns 0 results: should not happen because empty types are hidden from pills

### Loading
- Server-rendered page, no client-side loading state needed
- Patch list and detail pages are server components

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| < 640px | Single column cards, pills wrap to second line |
| >= 640px | Two column card grid, pills single row |

- Page padding: px-lg (24px) on all sizes
- Card grid max-width: 960px, centered
- Filter pills: flex-wrap on mobile, single-line with overflow-x-auto if many types

## Accessibility

- Filter pills use `role="tablist"` with `role="tab"` and `aria-selected`
- Active pill has `aria-selected="true"`
- Cards use semantic `<article>` elements within the grid
- Card links have descriptive aria-label: "Open patch: {patch name}"
- Type group headers use `<h2>` semantic level
- Parameter tables maintain existing accessible markup from .param-table
- Focus rings: 2px accent, offset-2, offset-bg (established pattern)
- Color contrast: #e8e8e8 on #0a0a0a = 17.4:1 (AAA), #737373 on #0a0a0a = 5.5:1 (AA), #c8ff00 on #0a0a0a = 15.3:1 (AAA)
