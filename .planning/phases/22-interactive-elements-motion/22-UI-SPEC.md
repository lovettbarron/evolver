## Motion Contract

### Spring Configuration

Personality: **understated and precise** (D-08). Low bounce, quick settle, Apple iOS-style springs.

| Spring Preset | Stiffness | Damping | Mass | Use Case |
|---------------|-----------|---------|------|----------|
| hover | 400 | 28 | 1 | Card hover lift + shadow transitions |
| celebration | 500 | 15 | 1 | Check icon entrance with overshoot |
| pulse | 300 | 20 | 1 | Button background color pulse on completion |

All spring values are Claude's discretion (CONTEXT.md). These targets produce snappy motion with minimal overshoot: hover settles in ~200ms, celebration overshoots once then settles in ~300ms.

### Card Hover — Spring Replacement (D-01)

Replaces Phase 21 CSS `transition-all 150ms` with motion springs on all five card types.

**Affected components:** HeroCard, PatchCard, ModuleCard, InstrumentCard, CountCard

```
/* Phase 21 CSS hover (REMOVED) */
transition: border-color 150ms, transform 150ms, box-shadow 150ms;

/* Phase 22 motion spring (REPLACES) */
motion.div wraps card content
  whileHover={{
    y: -2,
    boxShadow: "0 4px 12px oklch(0.85 0.18 105 / 0.06)"
  }}
  transition={{ type: "spring", stiffness: 400, damping: 28 }}
```

Border-color transition to accent on hover remains CSS (border-color change does not benefit from spring physics). Only `transform` and `box-shadow` move to spring.

**Resting state** (unchanged from Phase 21):
- border: 1px solid var(--color-border-subtle)
- background: var(--color-surface)
- border-radius: 8px
- y: 0, box-shadow: none

**Hover state** (same visual targets, spring physics):
- border-color: var(--color-accent) (CSS, instant)
- y: -2px (spring)
- box-shadow: 0 4px 12px oklch(0.85 0.18 105 / 0.06) (spring)

### Non-Card Spring Hover (D-02, Claude's Discretion)

| Element | Spring Hover | Rationale |
|---------|-------------|-----------|
| Nav links | No | Persistent UI, already has accent bottom bar active state |
| Buttons (CTA) | Yes — scale(1.02) on hover, spring stiffness 400 damping 28 | Buttons are primary interactive targets, subtle scale feels natural |
| SessionRow | No | List items should remain calm; bg-surface hover from Phase 21 is sufficient |

### Scroll Reveal (D-03, D-04, D-05)

**Animation:** Fade-up reveal. Elements start at opacity 0, translateY 16px. Animate to opacity 1, translateY 0.

**Stagger:** 50ms delay between siblings in grids and lists (D-04).

**Implementation approach** (Claude's discretion): Create a reusable `<ScrollReveal>` wrapper component using motion's `useInView` hook. This keeps animation logic centralized and testable.

```
<ScrollReveal>
  initial={{ opacity: 0, y: 16 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ type: "spring", stiffness: 300, damping: 25 }}
  viewport={{ once: true, margin: "-80px" }}
</ScrollReveal>
```

`viewport.once: true` — animate once on first entry, do not re-animate on scroll back.
`viewport.margin: "-80px"` — trigger 80px before element enters viewport so animation completes by the time element is fully visible.

**Stagger wrapper** for grids/lists:

```
Parent container uses motion staggerChildren: 0.05 (50ms)
Each child uses ScrollReveal with variants inheriting stagger delay
```

**Scroll reveal targets:**

| Component | Reveal Type | Stagger |
|-----------|-------------|---------|
| patch-grid.tsx | Card grid — each card fades up | Yes, 50ms between cards |
| module-index.tsx | Card grid — each module card fades up | Yes, 50ms between cards |
| session-list-client.tsx | List — each session row fades up | Yes, 50ms between rows |
| instrument-overview.tsx | Page sections — each section fades up | Yes, 50ms between sections |

**Not scroll-revealed** (D-05): Navigation bar, footer, completion toggle bar, page headings/breadcrumbs. These are persistent or above-the-fold UI that must appear instantly.

### Completion Celebration (D-06, D-07)

**On complete (checking):**
1. Checkbox background changes to --color-accent (existing behavior)
2. Check icon enters with spring overshoot: scale from 0 to 1, stiffness 500, damping 15
3. Button background pulses accent color: opacity 0 to 0.15 to 0, duration 400ms, then settles
4. "Completed" label text color transitions to --color-accent (CSS, 150ms)

All animation is contained within the button bounds (D-06). No particles, no effects escaping the element.

**On uncomplete (unchecking) (D-07):**
1. Check icon fades out: opacity 1 to 0, duration 100ms (not spring — quick exit)
2. Checkbox reverts to border-only state: CSS transition 150ms
3. Label text transitions to --color-muted: CSS transition 150ms
4. No celebration, no spring, no pulse

### Reduced Motion (D-09)

`prefers-reduced-motion: reduce` disables ALL motion library animations:

```css
@media (prefers-reduced-motion: reduce) {
  /* Disable all motion library animations */
  *, *::before, *::after {
    /* motion library respects this automatically via useReducedMotion() */
  }

  /* Ensure CSS fallbacks are also disabled */
  .animate-pulse-glow {
    animation: none;
    border: 2px solid var(--color-accent);
  }
}
```

Implementation: Use motion's built-in `useReducedMotion()` hook at the app level. When true:
- Card hover: instant CSS transform (no spring), or no transform at all
- Scroll reveal: elements render in final state immediately (opacity 1, y 0)
- Completion celebration: no spring entrance, no pulse. Check icon appears instantly
- All elements appear in their final visual state with zero animation

## Interaction States

### Card Hover (spring-driven)

```
Resting:
  y: 0
  box-shadow: none
  border-color: var(--color-border-subtle)

Hover:
  y: -2    (spring: stiffness 400, damping 28)
  box-shadow: 0 4px 12px oklch(0.85 0.18 105 / 0.06)    (spring)
  border-color: var(--color-accent)    (CSS instant)

Exit hover:
  Returns to resting via spring (natural deceleration, no abrupt snap)
```

### Button Hover (CTA buttons only)

```
Resting:
  scale: 1

Hover:
  scale: 1.02    (spring: stiffness 400, damping 28)

Exit hover:
  Returns to scale 1 via spring
```

### Completion Toggle States

```
Unchecked resting:
  Checkbox: border-2 border-muted, bg transparent
  Label: "Mark complete", color --color-muted

Checking (animate):
  Checkbox: bg --color-accent (CSS 150ms)
  Check icon: scale 0 → 1 (spring: stiffness 500, damping 15, ~1 overshoot)
  Button bg: pulse accent at 0.15 opacity → 0 (400ms ease-out)
  Label: "Completed", color --color-accent (CSS 150ms)

Checked resting:
  Checkbox: bg --color-accent, Check icon at scale 1
  Label: "Completed", color --color-accent

Unchecking (animate):
  Check icon: opacity 1 → 0 (100ms linear)
  Checkbox: bg transparent, border-muted (CSS 150ms)
  Label: "Mark complete", color --color-muted (CSS 150ms)
  No celebration, no spring
```

### Scroll Reveal States

```
Before viewport entry:
  opacity: 0
  y: 16px

Entering viewport (once, -80px margin trigger):
  opacity: 0 → 1    (spring: stiffness 300, damping 25)
  y: 16px → 0        (spring)

After animation:
  opacity: 1, y: 0 (final state, no re-animation)
```

### Focus-Visible (unchanged from Phase 21)

```
*:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
}
```
