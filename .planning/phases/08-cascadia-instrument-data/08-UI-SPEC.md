## Module Card Specification

Each module card displays:

```
+-----------------------------------------------+
| [CATEGORY BADGE]              [jack_count] jacks|
|                                                 |
| Module Name                                     |
| Purpose line (1-2 lines, line-clamp-2)         |
|                                                 |
| [control_count] controls  [normal indicator]    |
+-----------------------------------------------+
```

- Category badge: uppercase 14px, accent color, matches patch-card type badge
- Category values: `sound-source`, `shaper`, `modulator`, `utility`
- Module name: 20px/700 (Heading 3)
- Purpose: 16px/400 text color, `line-clamp-2`
- Metadata row: 14px muted, control count and jack count from frontmatter
- Normal indicator: text "Has normals" in muted when `has_normals: true`, omitted otherwise
- Card surface: `bg-surface rounded-[6px] p-lg border border-transparent hover:border-accent transition-colors`
- Grid: `grid grid-cols-1 sm:grid-cols-2 gap-md`

## Module Detail Page Specification

Layout follows `InstrumentOverview` pattern:

```
max-w-[720px] mx-auto px-lg lg:px-xl py-2xl
```

Content sections rendered from markdown in order:
1. **Title** -- 36px/700, category badge below title in muted
2. **Purpose** -- prose paragraph
3. **What Makes It Special** -- prose section
4. **Controls table** -- uses existing `.param-table` styling (Control | Type | Range | Notes)
5. **Patch Points table** -- uses existing `.param-table` styling (Jack | Type | Normalled To | Notes)
6. **LEDs** -- prose section
7. **Normalled Connections** -- prose section, optionally with inline Mermaid diagram

Back navigation: link to `/instruments/[slug]/modules` above the title, styled as `text-muted text-sm hover:text-accent` with left arrow icon (ChevronLeft from lucide-react, 16px).

## Module Index Page Specification

Layout:

```
max-w-[960px] mx-auto px-lg lg:px-xl py-2xl
```

Sections:
1. **Page title**: "Modules" at 36px/700
2. **Subtitle**: instrument name + total module count in muted 14px
3. **Category filter pills** (optional, same pattern as patch type filter): `sound-source | shaper | modulator | utility | all`
4. **Module card grid**: grouped by category with category headers at 20px/700

Filter pill styling (from existing patch filter pattern):
- Active: `bg-accent text-bg font-bold`
- Inactive: `bg-surface text-text border border-transparent hover:border-accent`
- Pill size: `py-xs px-md rounded text-sm`

## Signal Flow Diagram Specification

The Mermaid diagram on the signal-flow page uses:
- Subgraph groupings: Sound Sources, Shapers, Modulators, Output
- Solid lines (`-->`) for normalled connections
- Dashed lines (`-.->`) for available (unpatched) connections
- Node styling: default Mermaid theme (already rendered by `MermaidRenderer`)
- Container: `.mermaid-placeholder` styling during SSR, hydrated by client-side MermaidRenderer

No custom Mermaid theme overrides needed -- existing MermaidRenderer handles dark background rendering.

## Accessibility

- Module cards are wrapped in `<Link>` elements with `aria-label="Open module: {module name}"`
- Category filter pills are `<button>` elements with `aria-pressed` state
- Tables use `<th scope="col">` for column headers
- Mermaid diagrams have `aria-label="Signal flow diagram"` on their container
- Back navigation uses `aria-label="Back to all modules"`
- Color contrast: accent #c8ff00 on bg #0a0a0a = 12.5:1 (passes AAA); muted #737373 on bg #0a0a0a = 5.1:1 (passes AA)
