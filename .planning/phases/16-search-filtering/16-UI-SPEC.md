## Component Inventory

### 1. SearchInput (nav bar)

**Location:** Embedded in `nav.tsx`, between InstrumentSwitcher and nav links.

**Anatomy:**
- Container: `w-[200px]` collapsed, `w-[320px]` on focus (transition-all 200ms)
- Background: `bg-surface` with `border border-transparent`, `border-muted/30` on hover, `border-accent` on focus
- Border radius: `rounded-full` (matches existing pill style)
- Height: `h-[36px]` (matches existing pill height)
- Padding: `pl-[36px] pr-md` (left padding for icon)
- Search icon (magnifying glass SVG, 16x16): positioned absolute left 12px, `text-muted`, `text-accent` on input focus
- Placeholder: "Search sessions & patches..." at 14px, `text-muted`
- Input text: 14px, `text-text`
- Clear button (x icon, 14x14): appears when query is non-empty, positioned absolute right 12px, `text-muted hover:text-text`

**Keyboard:**
- `/` global shortcut focuses the search input (only when no other input is focused)
- `Escape` clears query and blurs input
- `ArrowDown` from input moves focus to first dropdown result
- `ArrowUp/ArrowDown` navigates within dropdown results
- `Enter` on a result navigates to it

**States:**
- Empty (collapsed width, placeholder visible, no dropdown)
- Focused (expanded width, accent border, dropdown appears if query >= 1 char)
- With results (dropdown open, results grouped)
- No results (dropdown open, empty state message)

### 2. SearchDropdown (overlay)

**Location:** Absolutely positioned below SearchInput, same width as expanded input (320px).

**Anatomy:**
- Container: `bg-bg border border-muted/20 rounded-[8px] shadow-lg mt-xs`
- Max height: `max-h-[384px] overflow-y-auto`
- Category headers: 13px uppercase tracking-wider `text-muted` with count badge `text-accent`, `px-md py-xs`
- Result items: `px-md py-sm hover:bg-surface rounded-[4px] cursor-pointer`
- Result item focused (keyboard): `bg-surface outline-none ring-1 ring-accent/30`

**Session result item:**
- Format: `#{number} {title}` at 14px `text-text`
- Metadata: `{module name}` at 13px `text-muted`, right-aligned

**Patch result item:**
- Format: `{name}` at 14px `text-text`
- Badge: `[{type}]` at 13px uppercase `text-accent`, inline after name

**Category footer (conditional):**
- Shown when results > 4 per category
- Text: "See all {N} sessions" / "See all {N} patches" at 13px `text-accent hover:underline`, `px-md py-xs`
- Links to `/instruments/{slug}/sessions?q={query}` or `/instruments/{slug}/patches?q={query}`

**Dismiss:** Click outside, Escape key, or selecting a result.

### 3. PatchFilterBar (patches page)

**Location:** Replaces the existing type-only tablist in `PatchGrid`. Wraps the grid.

**Row 1 — Type pills + Sort:**
- Layout: `flex flex-wrap items-center gap-sm`
- Type pills: Exact existing style from PatchGrid (preserved). Active: `bg-accent text-bg font-semibold rounded-full px-md py-xs min-h-[36px] text-sm capitalize`. Inactive: `bg-transparent text-muted border border-muted/30 rounded-full px-md py-xs min-h-[36px] text-sm hover:text-text hover:border-text transition-colors capitalize`
- Multi-select behavior: clicking a pill toggles it on/off independently. Multiple types active = OR filter. If all are deselected, show all (same as "All" active). "All" pill deselects all specific types
- Sort dropdown: right-aligned with `ml-auto`. Trigger: `text-sm text-muted hover:text-text flex items-center gap-xs cursor-pointer`. Label: "Sort: {value}" with chevron-down SVG (12x12). Dropdown: `bg-bg border border-muted/20 rounded-[6px] shadow-md py-xs min-w-[120px]`. Options: `px-md py-xs text-sm hover:bg-surface cursor-pointer`. Active option: `text-accent`
- Sort options: "Name" (default), "Date", "Type"

**Row 2 — Tag pills (conditional):**
- Layout: `flex flex-wrap gap-xs mt-sm` (only renders when tags exist in filtered results)
- Tag pills: smaller than type pills. Active: `bg-accent/20 text-accent border border-accent/40 rounded-full px-sm py-[2px] text-[13px]`. Inactive: `bg-transparent text-muted border border-muted/20 rounded-full px-sm py-[2px] text-[13px] hover:text-text hover:border-muted/40 transition-colors`
- Tag pills prefixed with `#` character
- Tags shown: only tags present in currently type-filtered results

**Filter logic:**
- Type + Tag combine with AND (must match at least one active type AND at least one active tag)
- Within types: OR (matches any active type)
- Within tags: OR (matches any active tag)
- URL state: `?type=bass,lead&tag=dark,ambient&sort=date` — filters persist in URL params for shareability

### 4. FilterEmptyState (patches page)

**Location:** Replaces patch grid content when filters yield zero results.

**Anatomy:**
- Container: `text-center py-3xl`
- Icon: none
- Heading: 16px `text-text font-semibold` — contextual message (see Copywriting)
- Body: 14px `text-muted mt-sm`
- Action: `mt-md` — "Clear filters" button styled as text link: `text-accent hover:underline text-sm cursor-pointer`

## Interaction Contracts

### Search Debounce
- No debounce needed per D-14 (client-side, instant filtering). Filter on every keystroke.

### Focus Management
- On `/` keypress: focus search input (guard: `document.activeElement` is not an input/textarea)
- On `Escape` in input: if query exists, clear it; if query empty, blur input
- On result selection (Enter or click): navigate to result URL, dismiss dropdown, blur input
- On dropdown dismiss: return focus to search input

### Filter Persistence
- Patch filters (type, tag, sort) persist in URL search params
- Search query does NOT persist in URL (ephemeral overlay)
- Navigating away from patches page and returning preserves filter state via URL

### Animation
- Search input width transition: `transition-all duration-200 ease-out`
- Dropdown appear: no animation (instant, per Spotlight convention)
- Filter pill toggle: `transition-colors` (existing pattern)
