- Patch type badge (existing pattern from patch-detail)
- "Send to Device" button (accent outline variant)

## Layout Contract

### MIDI Page Structure

The MIDI page is a **workspace** (interactive tool), not a browse view. Single-column layout, max-width 960px, centered.

```
[Sticky Header: "MIDI" + back to instrument]
[Connection Section: device selector + status]
[Capture Section: capture button + listen toggle + preview]
[Send Section: patch selector + send button]
[Diff Section: picker + side-by-side view]
```

### Section Card Pattern

Each section (Connection, Capture, Send, Diff) uses a consistent card pattern:
- Background: `bg-surface`
- Border radius: 6px
- Padding: `p-lg` (24px)
- Margin bottom: `mb-xl` (32px)
- Section heading: 20px bold, `mb-md`

### Diff View Layout

- Two columns, equal width, inside the diff section card
- Gap between columns: `gap-md` (16px)
- Column header: patch name, 14px semibold
- Parameter rows: aligned by parameter name across both columns
- Section group headers (Oscillators, Filter, Envelopes, LFOs, Sequencer): 14px uppercase tracking-wider `text-muted`, sticky within scroll
- Scrollable area with max-height: `max-h-[600px]` with overflow-y-auto
- On screens below 768px: stack columns vertically, each full-width

### Connection Status Bar

- Inline within Connection section, not a global banner
- Layout: `flex items-center gap-sm`
- Status dot: 8px circle, `bg-accent` when connected, `bg-muted` when disconnected
- Device name in mono font when connected
- "No MIDI device detected" in muted text when disconnected

## Interaction States

### MIDI Connection

| State | Visual | Copy |
|-------|--------|------|
| No Web MIDI support | Yellow warning banner at top of page | "Your browser does not support Web MIDI. Use Chrome or Edge to connect your Evolver." |
| Detecting devices | Pulsing muted dot + muted text | "Detecting MIDI devices..." |
| No device found | Muted dot + muted text + manual select dropdown | "No MIDI device detected. Connect your Evolver and refresh, or select a device manually." |
| Connected | Accent dot + device name in mono | "Connected: [Device Name]" |
| Disconnected (was connected) | Muted dot + muted text | "Device disconnected. Reconnect your Evolver." |

### Capture Flow

| State | Visual | Interaction |
|-------|--------|-------------|
| Idle | "Capture Current Patch" button (accent bg, dark text) | Click triggers SysEx dump request |
| Capturing | Button disabled, shows "Receiving..." with subtle pulse animation | Non-interactive during capture |
| Received | Parameter preview appears below button in a scrollable param table | User reviews parameters |
| Save prompt | Inline form below preview: name input, type dropdown, optional tags | User fills and clicks "Save to Library" |
| Saved | Brief success message "Patch saved to library" in accent text, form clears | Returns to idle |
| Error | Error message in muted text below button | "Capture failed. Check MIDI connection and try again." |

### Listen Mode

- Toggle switch (pill-shaped, accent when active) labeled "Listen Mode"
- When active: muted helper text "Listening for incoming SysEx dumps..."
- Positioned to the right of the Capture button on the same row

### Send to Device

| State | Visual | Interaction |
|-------|--------|-------------|
| Idle | Patch dropdown selector + "Send to Device" button (accent outline) | Select patch, click send |
| Confirm | Modal overlay with dark backdrop | See confirmation dialog spec below |
| Sending | Button disabled, "Sending..." | Non-interactive |
| Success | Brief inline text "Patch loaded on Evolver" in accent | Auto-clears after 3 seconds |
| Error | Inline error text | "Send failed. Check MIDI connection and try again." |

### Confirmation Dialog

- Centered modal, max-width 400px
- Background: `bg-surface`, border-radius 6px
- Padding: `p-lg`
- Backdrop: `bg-bg/80` with blur
- Heading: "Send Patch to Evolver" at 20px bold
- Body: "This will load **[Patch Name]** into the Evolver's edit buffer. Your current unsaved edits on the hardware will be replaced."
- Buttons row: "Cancel" (muted text, no bg) + "Send Patch" (accent bg, dark text)
- Escape key and backdrop click dismiss

### Diff View

| State | Visual |
|-------|--------|
| No patches selected | Empty columns with muted placeholder text |
| One patch selected | That column populated, other shows "Select a patch" |
| Both selected, no differences | All rows in muted text, summary line "Patches are identical" |
| Both selected, with differences | Changed rows highlighted with accent, summary "N parameters differ" |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| MIDI status | `aria-live="polite"` on connection status region |
| Capture feedback | `aria-live="assertive"` on capture result messages |
| Confirm dialog | `role="dialog"`, `aria-modal="true"`, focus trap, Escape to dismiss |
| Diff table | Use `<table>` elements with proper `<th>` headers, not div grids |
| Buttons | All interactive elements min 44px touch target |
| Keyboard | Tab order follows visual layout. Enter/Space activate buttons. Escape closes dialog |
