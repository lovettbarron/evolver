## Color

All colors use the OKLCH token system from Phase 18. This phase extends the accent system to be fully instrument-adaptive.

| Role | Token | OKLCH Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | --color-bg | oklch(0.12 0.01 85) | Page background |
| Secondary (30%) | --color-surface | oklch(0.16 0.01 85) | Card backgrounds, heatmap empty cells |
| Accent (10%) | --color-accent | oklch(0.85 0.18 105) / oklch(0.75 0.12 185) | See reserved-for list below |
| Destructive | --color-destructive | oklch(0.55 0.2 25) | Not used in this phase |

Accent reserved for (this phase):
- Heatmap cell intensity fills (4 opacity levels via color-mix)
- Panel glow circle fill (replacing hardcoded #c8ff00)
- Card hover shadow tint (replacing hardcoded oklch lime in --shadow-card-hover)
- Pulse-glow keyframe shadow color (replacing hardcoded oklch lime)
- Focus-visible outline ring (inherited, already token-based)
- ModuleJourney current-module dot (verify already token-based)

Not accent: heatmap empty cells (use --color-surface), heatmap day labels (use --color-muted), panel SVG control internals (frozen per D-04).

### Heatmap Intensity Scale

4 levels plus empty, using color-mix with var(--color-accent):

| Level | Condition | Background |
|-------|-----------|------------|
| Empty | count = 0 | var(--color-surface) |
| Low | count = 1 | color-mix(in oklch, var(--color-accent) 25%, transparent) |
| Medium | count = 2 | color-mix(in oklch, var(--color-accent) 50%, transparent) |
| High | count = 3 | color-mix(in oklch, var(--color-accent) 75%, transparent) |
| Max | count >= 4 | var(--color-accent) at 100% |

## Motion Contract

### Panel ViewBox Tween (D-01, D-02, D-03)

Smooth spatial zoom from full panel view to focused section. No crossfade, no dimming -- viewport narrowing IS the contextual focus.

```
Auto-zoom sequence (session pages with zoomSections):
  1. Render with full viewBox (0 0 1200 520 for Evolver, 0 0 1000 580 for Cascadia)
  2. After ~200ms initial flash, tween viewBox to computed zoom target
  3. Tween: type "tween", duration 0.3s, ease "easeInOut"

Implementation:
  Replace <svg viewBox={viewBox}> with <motion.svg animate={{ viewBox: targetViewBox }}>
  transition={{ type: 'tween', duration: 0.3, ease: 'easeInOut' }}
```

**Reduced motion:** When prefers-reduced-motion is active, viewBox snaps instantly to target (no tween). motion library handles this automatically.

### No New Spring Configurations

This phase uses the existing motion infrastructure from Phase 22. The viewBox tween uses CSS easing (not spring) because spatial zoom reads better with smooth deceleration than with bounce.

## Interaction States

### Panel ViewBox Zoom

```
Initial (page load):
  viewBox: full panel dimensions
  All sections visible at natural scale

Auto-zoom (after ~200ms):
  viewBox: tweens to computed zoom target from SECTION_BOUNDS
  Duration: 300ms, ease: easeInOut
  Relevant section fills the viewport
  Non-relevant sections slide out of frame naturally

No zoom (no zoomSections prop):
  viewBox: stays at full panel dimensions
  No animation occurs
```

### Heatmap Cell States

```
Empty cell:
  background: var(--color-surface)
  border-radius: 2px
  No hover effect

Filled cell (1+ sessions):
  background: color-mix intensity per scale above
  border-radius: 2px

Hover (filled cell):
  title attribute tooltip: "{YYYY-MM-DD}: {N} session(s)"
  No visual hover change (keeps grid calm per ADHD constraint)
```

### Heatmap Empty State

```
When completionDates.length === 0:
  Section heading: "Practice Activity"
  Body: "Your practice heatmap appears here after your first completed session."
  No grid rendered -- clean, encouraging empty state
```

## ADHD 5-Second Test Contract (D-11, D-12)

Visual audit checklist for human verification. Each page must pass:

1. **Single focal point:** One clear visual hierarchy level draws the eye first
2. **No competing animations:** No auto-looping animations visible simultaneously. Stagger delays prevent overload. Background fades are acceptable
3. **Click count audit:** Same or fewer clicks to reach any destination compared to pre-v1.3 baseline
4. **Information density:** No page section requires more than 5 seconds of scanning to understand its purpose

This is a human verification task during UAT, not an automated test.
