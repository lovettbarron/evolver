    Props interface:
    ```typescript
    interface ResumeBarProps {
      instrumentSlug: string;
      vaultCompletions: number[];  // serialized from server Set (Sets can't cross server/client boundary)
      sessions: Array<{ slug: string; data: { session_number: number; title: string; module: string } }>;
      isDemo: boolean;
    }
    ```

    Implementation:
    1. Use `useHydrated()` — if not hydrated, return `null` (per UI-SPEC Interaction States: render nothing during SSR/hydration)
    2. Get manual completions: `useLearnerStore(s => s.getCompletedSessions(instrumentSlug))`
    3. Get lastVisited: `useLearnerStore(s => s.lastVisited[instrumentSlug])`
    4. Merge: `const merged = mergeCompletions(new Set(vaultCompletions), manualCompletions)`
    5. Compute next: `const next = computeNextSession(lastVisited, merged, sessions)`
    6. Determine state variant:
       - If `next === null` -> "all complete" variant (D-06)
       - If `lastVisited === undefined` and completions are empty -> "ready to start" variant (first session)
       - Otherwise -> "has next session" variant (D-05)

    Visual implementation per UI-SPEC Section 2 (Resume Bar):
    - Wrapper: `<Link>` wrapping the entire bar for click behavior
    - Background: `bg-surface rounded-lg p-lg` (24px padding = --spacing-lg)
    - Layout: `flex items-center gap-md justify-between`
    - Hover: `hover:bg-[#1e1e1e] transition-colors duration-150 cursor-pointer`

    **Variant: has next session (D-05)**
    - Left icon: `<BookOpen size={20} className="text-muted shrink-0" />`
    - Middle (flex-col):
      - Label: `<span className="text-sm text-muted">Continue where you left off</span>`
      - Title: `<span className="text-base text-text">Session ${next.sessionNumber}: ${next.title}</span>`
    - Right: `<ArrowRight size={20} className="text-accent shrink-0" />`
    - Link href: `/instruments/${instrumentSlug}/sessions/${next.slug}`
    - `aria-label="Continue to session ${next.sessionNumber}: ${next.title}"`

    **Variant: all complete (D-06)**
    - Left icon: `<CheckCircle size={20} className="text-accent shrink-0" />`
    - Middle (flex-col):
      - Label: `<span className="text-sm text-accent">All sessions complete</span>`
      - Link text: `<span className="text-base text-text hover:underline">Explore patches and challenges</span>`
    - Right: `<ArrowRight size={20} className="text-muted shrink-0" />`
    - Link href: `/instruments/${instrumentSlug}/patches`
    - `aria-label="All sessions complete — explore patches and challenges"`

    **Variant: no sessions visited (empty state)**
    - Left icon: `<Play size={20} className="text-accent shrink-0" />`
    - Middle (flex-col):
      - Label: `<span className="text-sm text-muted">Ready to start?</span>`
      - Title: `<span className="text-base text-text">Session ${first.sessionNumber}: ${first.title}</span>`
    - Right: `<ArrowRight size={20} className="text-accent shrink-0" />`
    - Link href: `/instruments/${instrumentSlug}/sessions/${first.slug}`
    - Use `sessions[0]` for first session data

    **Demo mode (D-07):** Resume bar renders normally but uses only `vaultCompletions` (synthetic data from server). Since the store is empty in demo mode (no writes), `manualCompletions` will be empty Set, and the merge result is just the vault data. No special demo branching needed in the component itself — the data flow handles it.
