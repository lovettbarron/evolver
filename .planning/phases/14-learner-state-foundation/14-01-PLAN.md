       ```typescript
       import { create } from 'zustand';
       import { persist } from 'zustand/middleware';

       interface LearnerState {
         completions: Record<string, number[]>;  // keyed by instrument slug
         lastVisited: Record<string, { sessionSlug: string; sessionNumber: number }>;
         toggleCompletion: (instrument: string, sessionNumber: number) => void;
         setLastVisited: (instrument: string, sessionSlug: string, sessionNumber: number) => void;
         getCompletedSessions: (instrument: string) => Set<number>;
       }
       ```
       - `toggleCompletion`: if sessionNumber is in the array, remove it; otherwise append it
       - `setLastVisited`: overwrite the entry for the given instrument
       - `getCompletedSessions`: return `new Set(get().completions[instrument] ?? [])`
       - Persist config: `name: 'evolver-learner-state'`, `version: 1`
       - Use Zustand v5 curried form: `create<LearnerState>()(persist(...))`

    3. Create `src/hooks/use-hydrated.ts`:
       ```typescript
       import { useEffect, useState } from 'react';
       export function useHydrated() {
         const [hydrated, setHydrated] = useState(false);
         useEffect(() => { setHydrated(true); }, []);
         return hydrated;
       }
       ```

    4. Write tests in `src/stores/learner-store.test.ts` covering all behaviors listed above. Use Vitest. Reset store state between tests using `useLearnerStore.setState({ completions: {}, lastVisited: {} })`.

    Anti-patterns to avoid (from RESEARCH.md):
    - Do NOT create a React context provider wrapper. Zustand does not need providers.
    - Do NOT import the store in server components.
