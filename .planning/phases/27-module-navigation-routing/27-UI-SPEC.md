**CSS pattern for each module** (follows existing instrument pattern exactly):

```css
[data-instrument="plaits"] {
  --color-accent: oklch(0.70 0.12 165);
  --color-param: oklch(0.78 0.08 170);
  --color-bg: oklch(0.12 0.02 165);
  --color-sunken: oklch(0.10 0.02 165);
  --color-surface: oklch(0.16 0.02 165);
  --color-surface-raised: oklch(0.20 0.02 165);
  --color-overlay: oklch(0.24 0.02 165);
  --color-border: oklch(0.25 0.02 165);
  --color-border-subtle: oklch(0.20 0.015 165);
}
```

Repeat the pattern for all 7 modules using their respective hue, accent L/C, and surface chroma values from the table above. The `--color-param` hue should be offset +5 from the accent hue, with lightness 0.78 and chroma at 60-70% of the accent chroma.

**WCAG AA compliance:** All accent colors have lightness >= 0.65 against surface backgrounds at lightness 0.16, producing contrast ratios above 4.5:1.

## Interaction Contracts

### Category Tabs (`ModuleCategoryTabs`)

- **State:** Query param `?category=<slug>` drives active tab. "All" tab clears the param (D-02)
- **Tabs:** All | VCO | Filter | Effects | Modulator | Function Generator
- **Style:** Pill-shaped buttons in a horizontal scroll row. Active tab: `bg-accent text-bg font-semibold`. Inactive: `bg-surface-raised text-muted hover:text-text`
- **Sizing:** Height 36px, horizontal padding 16px, border-radius `--radius-full` (9999px)
- **Spacing:** 8px gap between tabs, 16px bottom margin below tab row
- **Interaction:** Click updates URL query param. No page reload — `useSearchParams` + `router.push` with shallow navigation
- **Multi-category:** Modules with multiple categories (e.g., Maths = function-generator + modulator) appear under every matching tab (D-03)
- **Keyboard:** Tab key moves between pills. Enter/Space activates. Focus ring uses `--color-accent`

### Module Card (`ModuleCard`)

- **Layout:** Vertical card using existing `.card` class. Content stacked: HP outline placeholder, module name, manufacturer, category tags, stat line
- **HP Outline Placeholder** (D-05): Proportional-width rectangle at card top. Width = `(hp / 20) * 100%` of card width, capped at 100%. Height 48px. Background: `--color-accent` at 15% opacity. Border: 1px solid `--color-accent` at 30% opacity. Border-radius: `--radius-sm` (4px)
- **Module name:** 20px Space Grotesk bold, `--color-text`
- **Manufacturer:** 14px Inter regular, `--color-muted`
- **Category tags:** Inline row of small pill badges. 14px uppercase with letter-spacing 0.05em, `--color-accent` text, `bg-accent/10`, border-radius `--radius-full`, padding `4px 8px`
- **Stat line:** 14px JetBrains Mono, `--color-param`. Format: `{hp}HP / {sessionCount} sessions`
- **CTA:** "Explore {name}" link, 14px, `--color-accent`, underline, underline-offset 2px
- **Hover:** Border transitions to `--color-accent` (existing `.card:hover` behavior). Wrapped in `SpringCard` for spring-physics lift
- **Grid:** 1 column on mobile (<640px), 2 columns at sm (640px+), 3 columns at lg (1024px+). Gap: `--spacing-xl` (32px)

### Module Sub-Page Tabs (`ModuleSubNav`)

- **Position:** Below the persistent module header (name, manufacturer, HP, categories). Sticky at nav-height offset (top: 60px)
- **Tabs:** Overview | Sessions | Patches | Panel
- **Style:** Text links in a horizontal row, matching the existing nav sub-link pattern. Active tab: `text-text` with a 2px bottom `bg-accent` underline. Inactive: `text-muted hover:text-text`
- **Height:** 48px, centered vertically
- **Spacing:** 16px gap between tab labels
- **Background:** `--color-surface-raised` with bottom border `--color-border`
- **Keyboard:** Standard link navigation. Active state indicated by `aria-current="page"`

### Module Header (in layout)

- **Position:** Top of module layout, above sub-nav tabs
- **Content:** Module display name (h1, display font), manufacturer name (muted), HP badge (mono, param color), category tag row
- **Padding:** `--spacing-2xl` (48px) top, `--spacing-lg` (24px) bottom
- **Max width:** `--content-wide` (1200px), centered

### Nav "Modules" Link

- **Position:** After InstrumentSwitcher, before search bar. Visible on all breakpoints
- **Style:** Text link, 14px, matches existing nav link style. Active when pathname starts with `/modules`
- **Active indicator:** 2px bottom `bg-accent` bar (same as existing sub-links)
- **Mobile:** Appears in the hamburger menu dropdown alongside other links

## Accessibility

- All category tabs are keyboard-navigable with visible focus rings (`2px solid --color-accent`)
- Module cards are wrapped in `<Link>` elements with meaningful accessible names
- Sub-page tabs use `aria-current="page"` for active state
- Category filter tabs use `role="tablist"` and `role="tab"` with `aria-selected`
- Color contrast: All accent colors meet WCAG AA (4.5:1+) against their surface backgrounds
- Mobile touch targets: All interactive elements >= 44px in the smaller dimension
- Reduced motion: `SpringCard` already respects `prefers-reduced-motion`
