  <action>
1. Update `package.json` scripts — add prebuild hook:
   ```json
   "prebuild": "npm run bundle-content"
   ```
   This ensures `src/content/` is populated before `next build` runs on Vercel.

2. Update `next.config.ts` to add outputFileTracingIncludes (merge with any existing config from plan 01):
   ```typescript
   import type { NextConfig } from 'next';

   const nextConfig: NextConfig = {
     outputFileTracingIncludes: {
       '/*': ['./src/content/**/*'],
     },
     webpack: (config) => {
       config.resolve.alias.canvas = false;
       return config;
     },
   };

   export default nextConfig;
   ```
   The `outputFileTracingIncludes` tells Next.js Node File Tracer to bundle all files in `src/content/` (sessions, patches, instruments, references) into the serverless function output. Without this, dynamic `fs.readdirSync` paths in `reader.ts` won't be traced and deployed pages will show empty content.

3. Create `vercel.json` at project root:
   ```json
   {
     "$schema": "https://openapi.vercel.sh/vercel.json",
     "buildCommand": "npm run prebuild && npm run build",
     "framework": "nextjs"
   }
   ```
   The `buildCommand` chains prebuild (content bundling) with build (Next.js). Framework hint ensures Vercel uses the correct adapter.

4. Run a local build to verify the full chain works:
   ```bash
   npm run prebuild && npm run build
   ```
   This must succeed before attempting Vercel deploy.

5. Deploy to Vercel using the CLI:
   ```bash
   npx vercel --yes
   ```
   If Vercel CLI is not authenticated, this will become a checkpoint for the user to authenticate. If it succeeds, capture the deployment URL.

6. If deploy succeeds, also set up production deployment:
   ```bash
   npx vercel --prod --yes
   ```
