## Color

All colors use the OKLCH token system from Phase 18.

| Role | Token | OKLCH Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | --color-bg | oklch(0.12 0.01 85) | Page background |
| Secondary (30%) | --color-surface | oklch(0.16 0.01 85) | Card backgrounds, callout backgrounds |
| Accent (10%) | --color-accent | oklch(0.85 0.18 105) | See reserved-for list below |
| Destructive | --color-destructive | oklch(0.55 0.2 25) | Not used in this phase |

Accent reserved for (this phase only):
- Card hover border color (D-02, D-10)
- HeroCard accent-left bar or distinct emphasis element (D-03)
- Card hover shadow tint at ~0.06 opacity (D-10)
- Focus-visible outline ring on all interactive elements (D-05)
- `::marker` numeral color on ordered steps (D-08)
- Parameter pill left-border accent (D-07)
- "Start Session" button background on HeroCard

Not accent: card borders at rest (use --color-border-subtle), session row hover background (use --color-surface), section divider hr (use --color-border-subtle).

## Interaction States

### Card Hover (all card types except SessionRow)

```
/* Resting state */
border: 1px solid var(--color-border-subtle);
background: var(--color-surface);
border-radius: 8px;
transition: border-color 150ms, transform 150ms, box-shadow 150ms;

/* Hover state */
border-color: var(--color-accent);
transform: translateY(-2px);
box-shadow: 0 4px 12px oklch(0.85 0.18 105 / 0.06);
```

### SessionRow Hover

```
/* Resting state */
background: transparent;

/* Hover state */
background: var(--color-surface);
```

### Focus-Visible (global)

```
*:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
}
```

Per-component overrides (Claude's discretion from CONTEXT.md):
- Search input: consider inset focus (outline-offset: -2px) if outer outline clips container
- Completion toggle: inherit global unless visual testing shows conflict

### HeroCard Hero Treatment

Accent left bar approach (Claude's discretion resolved):
```
border-left: 3px solid var(--color-accent);
border-radius: 2px 8px 8px 2px;  /* sharp left edge for bar, rounded right */
```

This gives the HeroCard a distinct editorial "pull quote" feel that differentiates it from standard cards without requiring gradients or complex backgrounds.

## Accessibility Contract

| Requirement | Specification | WCAG |
|-------------|---------------|------|
| Focus visibility | 2px solid accent outline, 2px offset on all interactive elements via :focus-visible | 2.4.7 Focus Visible (AA) |
| Focus not on click | :focus-visible only (not :focus), so mouse clicks do not trigger outlines | Best practice |
| Card border contrast | --color-border-subtle (oklch 0.20 0.008 85) against --color-bg (oklch 0.12 0.01 85) provides visible boundary | 1.4.11 Non-text Contrast |
| Accent border on hover | --color-accent (oklch 0.85 0.18 105) against card surface provides strong 4.5:1+ contrast | 1.4.11 |
| Step marker color | Accent numerals are decorative (content conveyed by number itself); color alone not required for meaning | 1.4.1 Use of Color |
| Param pill readability | Mono text at 14px on surface-raised background; inherits existing WCAG-validated token contrast | 1.4.3 Contrast |
