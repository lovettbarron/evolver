## Icon Specifications

All icons are inline SVG, 16x16 viewport, rendered at 16px width/height. No external icon library.

### Prerequisite State Badge Icons (D-01)

**Completed** — Checkmark circle:
- 16x16 SVG, circle with checkmark inside
- Fill: `--color-accent` (#c8ff00)
- Stroke: none
- Semantics: `aria-label="Completed"`

**Available** — Open circle:
- 16x16 SVG, circle outline only
- Fill: none
- Stroke: `--color-muted` (#737373), 1.5px stroke width
- Semantics: `aria-label="Available"`

**Locked** — Lock:
- 16x16 SVG, padlock shape
- Fill: `--color-muted` (#737373)
- Stroke: none
- Semantics: `aria-label="Locked"`

### Placement in SessionRow

Badge renders left of the session number with `gap: 8px` (sm). The existing SessionRow layout changes from:

```
[number. title]                [duration]
```

to:

```
[icon 16px] [gap 8px] [number. title]    [duration]
            [         hint text        ]
```

When locked (D-02), hint text "Requires #NN" renders below the title line in `14px --color-muted`, indented to align with the title text (offset by 16px icon + 8px gap = 24px left padding).

## Component Inventory

### 1. SessionRow (modify existing)

**File:** `src/components/session-row.tsx`

New props:
- `state: 'completed' | 'available' | 'locked'`
- `prerequisiteNumber: number | null`

Visual changes:
- Add badge icon left of session number
- When `state === 'locked'`, show "Requires #NN" hint below title
- All states remain clickable (soft gating per D-04, D-06)

### 2. PrerequisiteBanner (new component)

**File:** `src/components/prerequisite-banner.tsx`

Renders on session detail page when the current session has an unmet prerequisite.

Layout:
- Full width within content area (max-width 640px inherits from page)
- Background: `--color-surface` (#161616)
- Border-left: 3px solid `--color-accent` (#c8ff00) — matches existing callout pattern
- Padding: 16px (md)
- Border-radius: 0 6px 6px 0 — matches existing callout pattern
- Margin-bottom: 16px (md)

Content:
- Body: "This session builds on Session #NN — complete it first for the best experience" (D-04)
- "Session #NN" is a link: `--color-accent` text, underline, navigates to prerequisite session (D-05)
- Dismiss button: "X" icon (16x16 SVG), right-aligned, `--color-muted`, hover `--color-text`

Dismissal behavior (Claude's Discretion decision):
- Dismiss per page visit (not persisted). Reappears on next navigation to the same session
- Uses client-side React state, not localStorage

Accessibility:
- `role="status"` on the banner container
- Dismiss button: `aria-label="Dismiss prerequisite notice"`

### 3. CountCard (modify existing)

**File:** `src/components/count-card.tsx`

New props:
- `href: string`

Visual changes (D-08):
- Wrap in Next.js `Link` component
- Add `cursor-pointer` on hover
- Hover state: `translateY(-2px)` + `box-shadow: 0 4px 12px rgba(200, 255, 0, 0.08)` — subtle accent-tinted shadow
- Transition: `transform 150ms ease, box-shadow 150ms ease`
- Focus-visible: 2px accent outline offset 2px (keyboard accessibility)

### 4. ModuleJourney (modify existing)

**File:** `src/components/module-journey.tsx`

New prop on module items:
- Third state: `'current'` in addition to `complete: boolean` (becomes `state: 'complete' | 'current' | 'future'`)

Visual states for dots (12x12px circles):
- **Complete:** Solid fill `--color-accent` (#c8ff00) — existing behavior
- **Current (D-09):** Solid fill `--color-accent` (#c8ff00) + pulsing glow animation
- **Future:** `--color-surface` fill with 1px `--color-muted` border — existing "incomplete" behavior

Pulsing animation (Claude's Discretion decision):
```css
@keyframes pulse-glow {
  0%, 100% { box-shadow: 0 0 0 0 rgba(200, 255, 0, 0.4); }
  50% { box-shadow: 0 0 0 6px rgba(200, 255, 0, 0); }
}
```
- Duration: 2s
- Iteration: infinite
- Timing: ease-in-out
- Applied only to the dot with `state === 'current'`

When all modules complete (D-11): no pulsing dot, all dots solid accent.

### 5. CumulativeMetrics (new component)

**File:** `src/components/cumulative-metrics.tsx`

Renders on the progress page below the module journey section (D-12).

Layout:
- Section heading: "Practice Activity" — 24px, weight 700, `--color-text`
- Margin-top: 32px (xl) from module journey
- Margin-bottom: 16px (md) from heading to metrics
- Two metric cards side by side: `grid grid-cols-2 gap-md`

Each metric card:
- Background: `--color-surface` (#161616)
- Padding: 24px (lg)
- Border-radius: 8px (rounded-lg)
- Value: 24px, weight 700, `--color-accent` — e.g. "7"
- Label below value: 14px, weight 400, `--color-muted` — e.g. "Sessions this month"
- Margin between value and label: 4px (xs)

Metrics displayed (D-12, D-13):
1. **Sessions this month** — count of sessions marked complete with a completion date in the current calendar month (Claude's Discretion: calendar month, not rolling 30 days — simpler, resets naturally)
2. **Active weeks** — count of distinct ISO weeks containing at least one completion event

Tone (D-14): Additive-only. No comparisons, no streaks, no "you missed" language. Just counts that go up.

## Interaction Contracts

### Count Card Click (D-07)

| Card Label | Destination |
|------------|-------------|
| Sessions Completed | `/instruments/[slug]/sessions` |
| Patches Created | `/instruments/[slug]/patches` |
| Modules Done | `/instruments/[slug]/modules` |
| Challenges Completed | `/instruments/[slug]/sessions` (same as sessions) |

### Prerequisite Banner Dismiss

- Trigger: Click X button
- Effect: Banner fades out (opacity 1 to 0 over 150ms, then `display: none`)
- Scope: Current page visit only, reappears on re-navigation
- No localStorage persistence

### Locked Session Navigation (D-04, D-06)

- Tapping a locked session navigates normally (no blocking)
- Session detail page renders PrerequisiteBanner at top of content
- Banner is informational only — full session content visible below it

### Module Journey "You Are Here" (D-09, D-10)

- Current module = module containing the first incomplete session in sequence order
- Pulsing stops when user completes all sessions in that module (dot becomes solid)
- Next incomplete module's dot begins pulsing
- When all complete: all dots solid, no animation

## Accessibility

| Element | Requirement |
|---------|-------------|
| Session badge icons | `aria-label` on each: "Completed", "Available", "Locked" |
| Locked hint text | Visible text, no additional ARIA needed |
| Prerequisite banner | `role="status"` for screen reader announcement |
| Banner dismiss | `aria-label="Dismiss prerequisite notice"` |
| Count cards | Existing `aria-label` pattern preserved, Link wrapping adds keyboard focusability |
| Count card focus | `focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-2` |
| Pulsing dot | `aria-current="true"` on the current module dot |
| Cumulative metrics | Standard semantic HTML, no special ARIA needed |
| Reduced motion | Pulsing animation respects `prefers-reduced-motion: reduce` — replace animation with static accent border (2px solid accent) |
