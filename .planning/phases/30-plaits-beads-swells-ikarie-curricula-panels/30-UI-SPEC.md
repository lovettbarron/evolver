## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 (Inter) | 1.6 |
| Label / Muted | 13px (0.8125rem) | 400 (Inter) | 1.5 |
| Subheading (h3/h4) | 20px (1.25rem) | 700 (Inter) | 1.3 |
| Display (h1/h2) | clamp(1.5rem, ..., 2.4375rem) | 700 (Space Grotesk) | 1.15 |

Four sizes total: 13px, 16px, 20px, Display clamp (24px-39px).

**Source:** globals.css `.prose` rules, Phase 19.

### Phase-Specific Typography Notes

- **Controls reference table:** Use `.param-table` class for parameter/control listings -- 13px body text, 13px headers, mono values, accent left border.
- **Session frontmatter metadata:** 13px muted text (`.obsidian-tag` pattern).
- **Panel control labels:** Handled by SVG text elements within panel data files -- not governed by CSS typography tokens. Use 10-12px sans-serif within SVG viewBox.
- **Plaits mode names:** Use 13px mono for synthesis mode names in mode-pair session headings (e.g., "VA Oscillator + Waveshaper").

Primary focal point: the module name + tagline in the Hero section; panel thumbnail (when available) as secondary visual anchor.

4. **Basic Patch (Init State)** -- Specific control positions producing a known starting state
5. **Sessions** -- Ordered session list linking to individual sessions

### Panel Data Files (4 new files, not components)

| File | HP | Approx Controls | Pattern |
|------|----|-----------------|---------|
| `src/lib/plaits-panel-data.ts` | 12 | 25-28 | Data-driven (knobs + attenuverters + buttons + jacks + 16 LEDs) |
| `src/lib/beads-panel-data.ts` | 14 | ~30 | Data-driven (knobs + attenuverters + buttons + jacks + LEDs + CV assign) |
| `src/lib/swells-panel-data.ts` | 20 | 40-45 | Data-driven (sliders + knobs + buttons + switches + jacks) -- largest panel |
| `src/lib/ikarie-panel-data.ts` | 8 | 17-20 | Data-driven (knobs + fader + switch + jacks + trimmer) -- smallest panel |

Each data file exports:
- `CONTROL_METADATA` -- Record of control ID to type/name/signalType
- `CONTROL_POSITIONS` -- Record of control ID to `{ x, y }` (hand-placed from reference)
- `SECTION_BOUNDS` -- Record of section name to `{ x, y, width, height }`
- `midiToRotation()` -- Standard knob rotation helper

Control ID convention: `{type}-{slug}-{name}` e.g., `knob-plaits-frequency`, `jack-beads-in-l`, `slider-swells-decay`

**Note:** The generic `ModulePanel` renderer (Phase 28, PANEL-01) must ship before these data files produce visible output. Data files are authored now so they are ready.
