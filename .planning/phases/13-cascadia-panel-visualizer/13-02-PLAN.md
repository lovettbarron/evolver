3. **useKnobDrag hook** — Copy from evolver-panel.tsx. Same PointerEvent API with setPointerCapture guard for jsdom compat, 3px per MIDI unit sensitivity, vertical drag.

4. **useSliderDrag hook** — NEW. Similar to useKnobDrag but maps pointer Y position directly to slider value (0 at bottom, 127 at top) instead of using rotation. Calculate value from drag delta: `Math.round(-delta / 3)` same sensitivity. Clamp 0-127.

5. **computeZoomViewBox** — Copy from evolver-panel.tsx. Takes section names, looks up SECTION_BOUNDS, computes union bounding box with 20px padding.

6. **CABLE_COLORS constant:**
   ```typescript
   const CABLE_COLORS: Record<string, string> = {
     audio: '#ff6644',
     cv: '#3388ff',
     modulation: '#ffaa33',
     default: '#888888',
   };
   ```

7. **CablePath component** — Internal, renders a single bezier cable (per RESEARCH code example):
   - Look up source and destination positions from CONTROL_METADATA positions within SECTION_BOUNDS
   - Compute control point: midX = (src.x + dst.x) / 2, droop = min(80, 30 + abs(dx) * 0.15), midY = max(src.y, dst.y) + droop
   - Render `<path d="M x1,y1 Q midX,midY x2,y2" fill="none" stroke={color} strokeWidth={3} strokeOpacity={0.8} strokeLinecap="round" style={{ pointerEvents: 'stroke' }}>`
   - Include `<title>` child with "Source -> Dest: Purpose" for native tooltip

8. **KnobGroup component** — Memo'd. Same as evolver-panel.tsx but uses a single knob radius (r=12 for standard, no large/small distinction on Cascadia). Indicator line from center outward, rotated by midiToRotation(value).

9. **SliderGroup component** — NEW Memo'd. Renders vertical slider track (rect) with movable thumb:
   - Track: 6px wide rect, dark fill (#111), stroke #444
   - Thumb: 12px wide, 8px tall rect positioned at midiToSliderPosition(value) * trackHeight from bottom
   - Label below track in 5.5px text
   - Highlight: rect outline with glow filter when highlighted

10. **SwitchGroup component** — Copy from evolver-panel.tsx. Display-only toggle (per D-10).

11. **JackGroup component** — NEW. Renders a single patch jack as a circle:
    - Output jacks: r=6, fill #2a2a2a, stroke #666 1px
    - Input jacks: r=6, fill #1a1a1a, stroke #555 1px
    - Label below in 4.5px text
    - When highlighted: glow filter applied
    - ID attribute set for DOM lookup by PanelTooltip

12. **SVG filter defs** — Copy glow-blue and glow-amber from evolver-panel.tsx.

13. **Main CascadiaPanel component** — Memo'd:
    - Default viewBox `0 0 1800 350`
    - Compute effective viewBox from zoomSections via computeZoomViewBox
    - Uncontrolled mode: internal useState for knobValues when prop not provided
    - Build highlight lookup Set and color map from highlights prop
    - Render in order:
      a. Panel background rect (fill #111)
      b. Section tint rects for activeSections
      c. Section divider lines (dashed #333)
      d. Section labels (7px bold, #999)
      e. All controls by module: knobs, sliders, switches, jacks, LEDs
      f. Cable paths (rendered LAST for highest z-order per UI-SPEC)
    - Event delegation for hover: onMouseOver/onMouseLeave on SVG root with findControlId walker (same pattern as evolver-panel.tsx)
    - svgRef for tooltip positioning
    - Wrap SVG in `<div className="relative">` with PanelTooltip sibling

**SVG layout approach:** Position each module's controls within their SECTION_BOUNDS area. Within each module, arrange controls in a grid from top to bottom: section label at top, then knobs/sliders in rows, jacks at bottom. This matches the Cascadia's physical layout where jacks are below the knobs/sliders on each module.

**Control positions:** For each control in CONTROL_METADATA, compute absolute x,y position from:
- Module's SECTION_BOUNDS (x, y origin)
- Control's relative position within module (computed from control index within that module)
- Standard spacing: knobs 30px apart, sliders 22px apart, jacks 20px apart

The JACK_POSITIONS lookup should be a computed constant (not in CONTROL_METADATA to keep data layer clean): derive from SECTION_BOUNDS + per-module jack layout. Store as `Record<string, {x: number, y: number}>` for cable path lookups.
