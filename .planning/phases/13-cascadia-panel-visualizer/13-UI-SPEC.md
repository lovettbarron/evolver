### Cable Color Scheme (New for Cascadia)

Cables are the primary visual differentiator from the Evolver panel. Color encodes signal type for fast comprehension.

| Signal Type | Cable Color | Opacity | Usage |
|-------------|-------------|---------|-------|
| Audio | #ff6644 | 0.8 | Audio-rate connections (VCO out, mixer, VCA, etc.) |
| CV / Gate | #3388ff | 0.8 | Control voltage and gate signals |
| Modulation | #ffaa33 | 0.8 | LFO, envelope, S&H outputs to mod destinations |
| Default / untyped | #888888 | 0.6 | Fallback when signal type is not specified |

Cable rendering: Quadratic bezier curves (`<path d="M x1,y1 Q cx,cy x2,y2">`) with 3px stroke width, round linecap, no fill. Control point offset vertically by 40-60px below the straight-line midpoint to create a natural droop (gravity sag). This mimics physical patch cables.

Cable hover: stroke-width increases to 4px, opacity to 1.0, and a tooltip appears with "Source -> Destination: Purpose" text.

## Interaction Contracts

### Knob Drag (carried from Phase 12)

- Vertical drag: pointer down captures, drag up increases value (0-127), drag down decreases
- Sensitivity: 3px per unit (same as Evolver: `Math.round(delta / 3)`)
- Rotation range: -135deg (value 0) to +135deg (value 127)
- Cursor: `grab` at rest, `grabbing` while dragging

### Jack Hover

- On hover: jack stroke changes to highlight color (blue or amber based on `highlights` prop), tooltip appears
- Tooltip content: jack name, module, signal type (input/output)
- Tooltip position: above the jack via `getBoundingClientRect` (same as Evolver knob tooltips)

### Cable Hover

- On hover over cable path: stroke-width 3px -> 4px, opacity to 1.0
- Tooltip appears at cursor position showing: "Source -> Destination: Purpose"
- Hit area: use `pointer-events: stroke` with transparent wider stroke for easier targeting

### Section Zoom (carried from Phase 12)

- `data-zoom` attribute crops viewBox to specified section bounds
- `data-sections` without explicit `data-zoom` auto-zooms to those sections
- `data-zoom="false"` shows full panel

### Cable Visualization (new)

- `data-cables` attribute format: `"source>destination:type,source>destination:type"`
  - Example: `data-cables="vco-a-saw>mixer-in-1:audio,lfo-xyz-x>vco-a-fm:mod"`
- When `data-cables` is set, connected jacks automatically receive highlight glow
- Cable paths render on top of all other panel elements (highest z-order in SVG)

## Marker Format (Markdown Embedding)

Session authors embed panel diagrams using this HTML marker in markdown content:

```html
<div data-cascadia-panel
  data-sections="vco-a,mixer"
  data-knobs="knob-vco-a-freq:64,knob-mixer-level:100"
  data-highlights="jack-vco-a-saw:amber,jack-mixer-in-1:blue"
  data-cables="jack-vco-a-saw>jack-mixer-in-1:audio"
></div>
```

Attribute parsing rules:
- `data-sections`: comma-separated section names from SECTION_BOUNDS keys
- `data-knobs`: comma-separated `controlId:value` pairs (value 0-127)
- `data-highlights`: comma-separated `controlId:color` pairs (color is `blue` or `amber`)
- `data-cables`: comma-separated `sourceJackId>destJackId:signalType` triples
- `data-zoom`: optional override; omit for auto-zoom to sections, set `"false"` for full view
