## Interaction Contracts

### Related Sessions Card

- **Trigger**: Renders automatically when session frontmatter contains `cross_references` OR when the session is referenced by another session (auto-bidirectional at build time)
- **Position**: Below session prose content, above the `CompletionToggle` fixed bar
- **Layout**: Full-width card at `--content-narrow` (720px max), containing a vertical list of reference rows
- **Each reference row**: Clickable link. Left side: module name in `--color-accent` (14px, uppercase) + session title (16px). Right side: reason text in `--color-muted` (14px). On hover: background transitions to `--color-surface-raised` (150ms)
- **Empty state**: Component does not render (no card shell shown if zero references)
- **Heading**: "Related Sessions" at 20px / 700 weight, inside the card at top

### Category Suggestions

- **Trigger**: Renders on module overview pages when other modules share at least one category
- **Position**: Bottom of module overview page, after session list
- **Layout**: Each category group has a heading ("Other VCOs", "Other Modulators") followed by a grid of `InstrumentModuleCard` components (existing card pattern)
- **Grid**: `grid-cols-1 sm:grid-cols-2` with `gap-md` (16px)
- **Heading per group**: 14px uppercase `--color-accent` with `letter-spacing: 0.05em`, `margin-bottom: --spacing-sm` (8px)
- **No deduplication**: A module appearing in multiple categories shows in each group (per D-07)
- **Empty state**: Section does not render if no other modules share any category
- **Section top spacing**: `--spacing-2xl` (48px) separator from preceding content

### Module Progress Bar

- **Dimensions**: Height 6px, full width of parent container, border-radius `--radius-full` (9999px)
- **Track**: `--color-surface` background
- **Fill**: `--color-accent` (inherits per-module color), width as percentage of completed/total, minimum 2px when > 0 completed
- **Label**: Below the bar, right-aligned: "{N}/{M} sessions" in 14px `--color-muted`
- **Animation**: Fill width transitions via CSS `transition: width 300ms ease-out`
- **Reduced motion**: No change needed (CSS transition, not keyframe animation)
- **Placement**: On module overview page, between the module description and the session list

### Prerequisite Badges (Module Sessions)

- **Pattern**: Identical to existing instrument session prerequisite system
- **States**: `completed` (filled accent circle + checkmark), `available` (open muted circle), `locked` (lock icon + "Requires #N" subtitle)
- **Soft gating**: Locked sessions are still clickable (navigable) -- visual indication only, not access control
- **Source**: Reuse `getSessionState()` from `prerequisite.ts` and `SessionRow` component unchanged

### Demo Mode (Synthetic Journeys)

- **Maths journey**: ~8 sessions completed over 6 weeks, ADHD-paced (enthusiastic start, mid-break, return). Currently partway through curriculum
- **Plaits journey**: ~6 sessions completed over 5 weeks, similar pacing. Shows a second module at different progress level
- **Data shape**: Same `Set<number>` + `journey_weeks` array pattern as Evolver/Cascadia/Octatrack exports
- **Detection**: Existing demo mode detection in `config.ts` -- no new mechanism needed
- **Visual result**: Demo visitors see realistic progress bars, completed/available/locked badges, and resume bar state for Maths and Plaits
