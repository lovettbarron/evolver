    **cross-references.ts** — New file implementing build-time bidirectional resolution (per D-04):
    ```typescript
    export interface ResolvedReference {
      moduleSlug: string;
      sessionSlug: string;
      sessionTitle: string;
      moduleName: string;
      reason: string;
    }

    export async function buildCrossReferenceMap(
      config: AppConfig
    ): Promise<Map<string, ResolvedReference[]>>
    ```
    Algorithm:
    1. Call `discoverModules(config)` to get all module slugs
    2. For each module, call `listSessions(moduleSlug, config)` to get sessions
    3. For each module, call `loadModuleConfig(moduleSlug, config)` to get display_name
    4. Collect forward refs from `session.data.cross_references` (if present)
    5. For each forward ref A->B, also record reverse ref B->A
    6. Deduplicate by `moduleSlug/sessionSlug` key per target session
    7. Return Map keyed by `"moduleSlug/sessionSlug"` -> ResolvedReference[]

    **category-suggestions.ts** — New file implementing category grouping (per D-05, D-06, D-07):
    ```typescript
    export interface CategoryGroup {
      category: string;
      categoryLabel: string;  // "Other VCOs", "Other Filters", etc.
      modules: Array<{
        slug: string;
        name: string;
        tagline: string;
        sessionCount: number;
      }>;
    }

    export async function buildCategorySuggestions(
      currentModuleSlug: string,
      config: AppConfig
    ): Promise<CategoryGroup[]>
    ```
    Algorithm:
    1. Call `discoverModules(config)` + `loadModuleConfig` for all modules
    2. Get current module's categories array
    3. For each category, find other modules with the same category
    4. Format category label: capitalize, add "Other " prefix (e.g., "vco" -> "Other VCOs", "filter" -> "Other Filters", "function-generator" -> "Other Function Generators")
    5. No deduplication across groups (per D-07)
    6. Filter out groups with zero other modules
    7. For each module in suggestions, count sessions via `listSessions`
