## Copywriting Contract

Phase 11 is primarily content authoring (markdown sessions and patches) with one small UI verification. The copywriting contract covers the UI-visible elements.

| Element | Copy |
|---------|------|
| Primary CTA | "Explore {InstrumentName}" (existing on InstrumentCard, no change) |
| InstrumentCard counts | "{N} sessions / {N} patches" (existing format, counts update automatically) |
| Empty state heading | not applicable (Cascadia content will exist after this phase) |
| Empty state body | not applicable |
| Error state | not applicable (no new error states introduced) |
| Destructive confirmation | not applicable (no destructive actions in this phase) |

### Session Content Copywriting Standards

These standards apply to the 16 new markdown sessions:

| Element | Convention | Source |
|---------|-----------|--------|
| Session title format | "Session {N}: {Topic Name}" | Phase 10 established pattern |
| Knob value notation | Percentage (e.g., "50%", "fully CW") | D-07, Phase 10 format |
| Cable reference format | "MODULE JACK OUT to MODULE JACK IN" | Phase 9 cable routing schema |
| Callout type for tips | `> [!tip]` | Phase 10 sessions |
| Callout type for Cascadia-unique | `> [!info] Cascadia-Unique` | Phase 10 sessions |
| Callout type for normalling | `> [!warning] Normalled Connection` | Phase 10 sessions |
| Callout type for challenges | `> [!challenge]` | Phase 5 challenge callout system |
| Output checklist format | Markdown task list with `- [ ]` items | Phase 10 sessions |

### Patch Content Copywriting Standards

These standards apply to recipe session patches (Module 7):

| Element | Convention | Source |
|---------|-----------|--------|
| Patch name | Title case, descriptive (e.g., "Deep Sub Bass", "Cascadia Pad") | Phase 9 demo patches |
| Patch description | One sentence, describes the sound character | Phase 9 demo patches |
| Cable purpose | Present tense verb phrase (e.g., "Provides pitch CV", "Modulates filter cutoff") | Phase 9 cable routing schema |
| Knob value format | Clock position ("12 o'clock", "3 o'clock") or percentage ("75%") | D-07, Phase 10 format |

## Phase-Specific Notes

1. **Content-dominant phase:** 95% of this phase is markdown content authoring. The UI contract is inherited from Phases 1-10 with zero visual changes.

4. **InstrumentCard counts:** Already implemented with `{sessionCount} sessions / {patchCount} patches` in `--color-param` mono font. Counts update automatically when content is bundled.
