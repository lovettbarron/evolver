**Step 3: Props interface and uncontrolled mode**
```typescript
interface EvolverPanelProps {
  knobValues?: Record<string, number>;      // MIDI 0-127 per control ID
```

**Uncontrolled mode (CRITICAL):** When `knobValues` prop is omitted, the component MUST use internal state to manage knob values. This enables the standalone route and quick-ref panel to render without a parent providing state:
```typescript
const [internalValues, setInternalValues] = useState<Record<string, number>>({});
const effectiveValues = knobValues ?? internalValues;
const effectiveOnChange = onKnobChange ?? ((id: string, val: number) => {
  setInternalValues(prev => ({ ...prev, [id]: val }));
});
```
When a knob has no value in `effectiveValues`, default to `midiToRotation(64)` (0deg, noon position). This means all knobs start at noon in uncontrolled mode.

**Step 4: Knob rotation rendering (per D-02)**
For each knob `<g>` group, compute the indicator `<line>` rotation:
- If `effectiveValues[controlId]` exists, use `midiToRotation(effectiveValues[controlId])`
- Otherwise, use `midiToRotation(64)` (noon position, neutral default)
- Apply as `transform={`rotate(${rotation})`}` on the indicator `<line>` element

```typescript
function useKnobDrag(
  controlId: string,
  currentValue: number,
  onChange?: (controlId: string, value: number) => void,
) {
  // useState for isDragging
  // useRef for startY, startValue
  // onPointerDown: preventDefault, setPointerCapture, record start position
```
