# Quick Task 260419-0b2: Fix Ikarie Panel Layout to Use Diagonal Resonance Fader

Diagonal RESONANCE fader rendering via SVG rotate transform with adjustable DIAGONAL_SLIDER_ANGLE constant, plus removal of invented section labels

## What Changed

### DiagonalSliderComponent (new)

Created a dedicated `DiagonalSliderComponent` in `ikarie-panel.tsx` that wraps the fader track and thumb in a rotated `<g>` element while keeping the label outside the rotation group so it stays horizontal and readable. The component accepts an `angle` prop for the rotation.

### DIAGONAL_SLIDER_ANGLE constant

Added at the top of the constants section, set to `-35` (upper-left to lower-right diagonal). This is easily adjustable if the angle needs tuning after visual comparison with the physical panel.

### Render loop routing

The `case 'slider':` branch now checks if the control is `slider-ikarie-resonance` -- if so, it renders `DiagonalSliderComponent` with the diagonal angle. All other sliders continue to use the standard vertical `SliderComponent`, preserving backward compatibility.

### Removed invented labels

Removed three section labels ("CONTROLS", "MODULATION", "I/O") and the "FOLLOW normalled to MOD" annotation text. None of these appear on the physical Ikarie panel silk-screen. The divider lines remain for visual separation.
